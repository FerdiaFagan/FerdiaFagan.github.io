In a nutshell: What is this application


This application is composed of two parts:
  1. The browser plugin
  2. The native application (Java application)
As you search webpages through your browser, often we come across information we want to be able to find later. The "Google effect" is a term coined to refer to how humans are evolving to remember less of the actual information they read, and instead are remembering more where the information was found (and can be retrieved). This I think personally is great news. I believe that as AI gets better, humans will become more and more in danger of becoming obsolete by the tools we create... And so, a time will come that humans and machine will combine (in certain ways that are to abstract currently to explain in a nut shell.).
Here is an interesting paper and video about this "Google effect".

I personally am most interested in the combining of humans and machines in respect to how we retrieve information and remember things.

So what this application does is:
  1. Index as your browse the internet (i.e. your history) -> Similar to how Google scrape web pages for information, so you can search things. Just to put your mind at ease: this application will not steal any of your data. Everything (including the index) remains on your machine.
  2. Keeps track of your search history, so can search through webpages you looked at in the past.
  3. Allow users to tag and bookmarks webpages (i.e. annotate webpages with metadata) to then be searched through -> You will see what I mean by this if you watch the demo video.
  4. Allow users to create what I called "auto annotators", which given hostnames will automatically tag webpages as you visit them.
  5. Allow users to then search through this index, filtering by dates, tags, bookmarks, and most importantly a query made of keywords. Then the user can view the results.
So yeah, thats the nutshell. I think you will get a better picture if you look at the demo video. I also just want to re-iterate -> I am not taking ANY of your data. The philosophy behind this project has a backbone of a belief that users (and society) should gain independance from big tech, and should not have to contribute YOUR data unless you want to. And you definatly should not be bribed into using a service that exploits your personal data just because the service is free.
But, don't get your hopes up. This is a project I built in 3 months as a demo. The search results will not be accurate. No competitor to Google... yet.

Things to keep in mind when using the application

  • You must spell query terms correctly. If you don't, it will not find that word in the index.
  • Don't be afraid to use long queries of many words. Let me know in the survey how it goes, and if it is very slow, etc.
  • Try out and change the settings and see if you prefer the customization of when to index (described at the end of the video).

Requirements:

  1. Firefox browser (please make sure to use an up-to-date version)
  2. Java: Use Java 11
  3. OS: Windows (Windows 8 or 10) or Ubuntu (linux)

Ubuntu installation:

  1. Unzip the application.
  2. Then (with sudo) run the installation script located at:
    KeepLivingInThePast_Application/linux_installer/install_on_linux.sh
  3. Then open Firefox browser and take the .xpi file located at:
    KeepLivingInThePast_Application/browser plugin/keep_living_in_the_past_man-2.0-fx.xpi
    and drag that file onto your Firefox browser window. Agree to the install and the plugin will install into your browser
  4. Use the application!
  5. Then give feedback after maybe a week of use. No need to dedicate time explicitly testing this. All I would ask is to install the browser plugin, and when you are doing work/research and browsing using Firefox, use the plugin and see if it helps you find the information you were looking for. Then after some use, fill out the survey linked on this webpage.
